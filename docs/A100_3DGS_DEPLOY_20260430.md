## 4. 环境注意事项

### 4.1 为什么不用系统默认 Python

A100 用户目录里原本有一套：

- `~/.local/lib/python3.10/site-packages/torch 2.11.0+cu130`

它会覆盖 micromamba env 里的 `torch`。所以运行 3DGS 时必须带：

```bash
export PYTHONNOUSERSITE=1
```

### 4.2 为什么需要自建 CUDA env

A100 系统自带 `CUDA 13.1`，但这次为了稳定编译 3DGS 扩展，实际使用的是 env 内的：

- `torch 2.5.1`
- `torch.version.cuda = 12.4`
- `nvcc 12.4.131`

### 4.3 Conda CUDA 头文件的坑

`nvidia` channel 这套包把部分头文件放在：

```text
$CUDA_HOME/targets/x86_64-linux/include
```

而不是传统的：

```text
$CUDA_HOME/include
```

因此在 env 里补了这几个符号链接：

```bash
ln -sfn $ENV/targets/x86_64-linux/include/nv $ENV/include/nv
ln -sfn $ENV/targets/x86_64-linux/include/cccl $ENV/include/cccl
ln -sfn $ENV/targets/x86_64-linux/include/cccl/thrust $ENV/include/thrust
ln -sfn $ENV/targets/x86_64-linux/include/cccl/cub $ENV/include/cub
ln -sfn $ENV/targets/x86_64-linux/include/cccl/cuda $ENV/include/cuda
```

没有这些链接时，`fused-ssim` 会报：

- `nv/target: No such file or directory`
- `thrust/complex.h: No such file or directory`
- `cuda/__cccl_config: No such file or directory`

### 5.2 训练 smoke test

```bash

cd /home/sari/3dgs_src/gaussian-splatting

  -s /home/sari/datasets/3dgs/db/drjohnson \
```

## 7. 下一步建议

如果要把它接到现场业务，建议按这个顺序做：

1. 先用 `VPR` 采集稳定园区的一批参考图像和定位元数据
2. 决定 3DGS 数据组织方式：按园区 / 地图版本 / 相机拆目录
3. 增加从 `VPR` 结果到 COLMAP/3DGS 输入目录的转换脚本
4. 在 A100 上做正式长训练，不再用 `--iterations 100`
5. 视业务需要决定是否接 `SIBR_viewers` 或网页侧预览
